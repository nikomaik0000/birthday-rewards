# AI Development Workflow

## Before Coding

1. Read CLAUDE.md.

2. Read AI_RULES.md.

3. Understand the current project.

4. Propose an implementation plan.

5. Wait for approval.

---

## During Development

- Keep changes small.

- Avoid large refactors.

- Modify only related files.

- Do not touch unrelated code.

- Remove dead code.

---

## Before Finishing

Run:

- TypeScript

- ESLint

- Build

Verify:

- No broken imports

- No unused code

---

## Deliverables

Provide:

1. Modified file list

2. Explanation of each change

3. Git patch (.diff)

4. Wait for approval

Do not assume changes are automatically applied.

---

## Scope Rules

One phase = One goal.

Never mix unrelated features into the same phase.

---

## Documentation

If functionality changes:

Update:

- CHANGELOG.md

- TODO.md

- ROADMAP.md (if needed)

---

## Git

Do not create commits.

Do not push.

The repository owner is responsible for Git operations.
