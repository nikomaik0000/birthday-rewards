# Birthday Rewards

## Before making any changes

Please read the following project documents first:

1. docs/AI_RULES.md
2. docs/PROJECT_STYLE.md
3. docs/DESIGN_SYSTEM.md
4. docs/AI_WORKFLOW.md
5. docs/ROADMAP.md

Do not start coding until you understand the project structure and current phase.

---

## Development Rules

Before modifying any files:

- Summarize your implementation plan in 5–10 bullet points.
- Wait for my approval before making changes.

---

## Scope Rules

- One Phase = One Goal.
- Do not mix unrelated features.
- Avoid large refactors.
- Modify only files related to the current task.
- If more than 10 files will be modified, explain why first and wait for approval.

---

## Code Quality

- Remove unused code.
- Remove unused imports.
- Do not leave commented-out code.
- Do not duplicate existing components.
- Reuse existing UI whenever possible.

---

## Build Verification

Before finishing, verify:

- TypeScript passes.
- ESLint passes.
- Production build succeeds.
- No broken imports.
- No dead code.

---

## Documentation

If functionality changes, update:

- docs/CHANGELOG.md
- docs/TODO.md

If the roadmap changes, also update:

- docs/ROADMAP.md

Do not modify other documentation unless necessary.

---

## Deliverables

Before stopping, always provide:

1. Implementation summary.
2. List of modified files.
3. Explanation of why each file was modified.
4. A git patch (.diff) containing all changes.

Then stop and wait for my approval.

Do not commit.

Do not push.

Git operations will be performed by the repository owner.
